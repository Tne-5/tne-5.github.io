This project classifies coral images as bleached or unbleached. The interesting part is not only the final score, but the engineering around it: avoid leakage, add a domain-relevant color signal, and inspect where the model is looking.

Final run: 0.969 AUC on the test split.

Problem

Bleaching is visually subtle. Healthy coral can appear pale under some lighting, and underwater images often contain borders, color casts, inconsistent crops, and duplicate or near-duplicate photos. A model can look strong while learning the wrong shortcut.

Pipeline

Model

The baseline architecture is ResNet18. I modified the first convolution so it accepts four channels instead of three: RGB plus saturation from HSV color space. Saturation is useful here because bleaching often corresponds to loss of color intensity, so it is a simple feature with a domain reason behind it.

ComponentReason
Grouped splitReduces leakage from visually similar or duplicate images.
Saturation channelMakes color loss explicit instead of relying only on RGB filters.
Masked attention poolingReduces the effect of invalid pixels and borders.
Heatmap outputsProvides a way to inspect model focus after training.

Training

I experimented with BCE and focal loss. The reason for focal loss was class imbalance and ambiguous samples: it increases the penalty on difficult examples and reduces the effect of easy ones that the model already handles. 

python src/coral_model.py --data-root data --epochs 50 --batch-size 32 --pretrained

Result

The final model reached 0.969 AUC on the test split. More importantly, the repository keeps the reusable training loop, transforms, model definition, preparation script, and output path for predictions/heatmaps while excluding private image data and generated artifacts.

Public repository: Tne-5/coral-bleaching-cv.

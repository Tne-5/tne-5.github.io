Coral Bleaching Computer Vision

A PyTorch image-classification pipeline for detecting coral bleaching, using a ResNet18 backbone modified with a custom saturation channel, attention pooling, and BCE/focal-loss training. The final run reached 0.969 test AUC.

Problem

Coral bleaching detection is visually subtle: lighting, underwater color casts, image borders, duplicates, and ambiguous labels can all mislead a model. The project goal was not only to classify images, but to produce a training pipeline with enough preprocessing, split control, and output inspection to trust the result.

Approach

I used a ResNet18 backbone and expanded the first convolution to accept a fourth channel: HSV saturation. This let the model learn from RGB appearance while keeping a domain-relevant bleaching signal available. The model also used masked attention pooling so feature focus could avoid border artifacts and produce heatmap-style inspection outputs.

image folders -> duplicate pruning -> resize/split
             -> 4-channel transforms -> ResNet18 + attention pooling
             -> BCE/focal loss -> validation AUC checkpoint
             -> test heatmaps + prediction CSV

Engineering details

The cleaned repository excludes the image dataset, trained weights, and generated heatmap folders. It keeps the reusable training/model code, data-preparation script, requirements, and documentation so the pipeline can be rerun with local data.

Evidence

  • 0.969 AUC on test data from final run.
  • Custom 4-channel input with saturation.
  • Attention pooling with valid-pixel mask.
  • Grouped train/validation/test split to reduce leakage.
  • Heatmap outputs for model inspection.

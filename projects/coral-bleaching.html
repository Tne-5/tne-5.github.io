Project brief

Not just a final score

Coral bleaching detection sounds like a normal image-classification task until the data starts pushing back. Underwater lighting changes the apparent color of healthy coral, black borders can become accidental model shortcuts, and near-duplicate images can inflate validation performance if the split is careless.

The practical target was a training pipeline that could be rerun, audited, and inspected, not a notebook with a single impressive metric.
4input channels: RGB plus HSV saturation as a domain-relevant color signal.
0.969test AUC reported by the final run.
3clean splits: train, validation, and test with leakage control in mind.

Data pipeline

Make the image folders trustworthy first

The repository separates data preparation from model training. The preparation step converts labels into a clean folder structure, removes obvious duplicates, standardizes image handling, and keeps generated outputs out of Git.

01Collect

Load labeled coral imagery from local data folders or CSV mappings.

02Clean

Prune duplicate files and normalize folder structure for reproducible runs.

03Transform

Resize, augment, and compute HSV saturation as a fourth input channel.

04Train

Fit a modified ResNet18 with BCE and focal-loss experiments.

05Inspect

Export predictions, metrics, and heatmap-style attention outputs.

Model design

A small architecture change with a domain reason

The model uses a ResNet18 backbone, but the first convolution is adapted to accept four channels. RGB carries normal visual appearance, while HSV saturation gives the model a direct signal related to bleaching and color loss. Masked attention pooling helps reduce the chance that borders or invalid pixels dominate the final representation. 

image folders

  -> RGB + saturation transforms
  -> ResNet18 first conv expanded to 4 channels
  -> masked attention pooling
  -> BCE / focal-loss training
  -> AUC, heatmaps, prediction CSV

Leakage control

Grouped splits reduce the chance that near-duplicates appear across train and test.

Domain feature

Saturation is explicit rather than hoping RGB filters discover the relevant bleaching cue alone.

Inspection path

Prediction CSVs and heatmaps make model behavior easier to review after training.

Evaluation

Strong test AUC, with visible model outputs

The final run reached a 0.969 AUC on the test split. The more important engineering habit is that the score is not left alone: the code also supports prediction exports and heatmap-style visual checks so the result can be interpreted rather than simply reported.

Public code

Clean repository structure

The public repository intentionally excludes the image dataset, trained weights, generated heatmaps, and local outputs. It keeps the reusable model code, data-preparation utility, requirements, and README so reviewers can understand the pipeline without downloading private or heavy artifacts.

Automated Twitter Complaint Chatbot

A prototype that filters likely airline complaints, routes them into issue categories, and returns support-style response templates.

Problem

Social support teams receive noisy, high-volume tweets. The useful automation target is triage: identify complaints, classify the issue, and suggest a response while leaving final responsibility to a human operator.

Approach

The pipeline cleans tweet JSON, imports records into MongoDB, deduplicates messages, scores sentiment with a Twitter RoBERTa model, and uses zero-shot classification for complaint routing.

raw tweet JSON -> cleaning + MongoDB import
              -> duplicate removal
              -> sentiment score
              -> zero-shot issue category
              -> response template

Boundaries

  • No API credentials are included
  • No tweet dataset is included
  • MongoDB URI is configurable
  • Models download from Hugging Face at runtime

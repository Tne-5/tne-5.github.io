Code Back to projects

This was an early applied NLP project: build a prototype that listens to airline-related tweets, identifies likely complaints, classifies the complaint type, and drafts a response template.

Problem

Social-media support is noisy. Many tweets mention airlines without needing action, and complaints can range from baggage to delays to refunds. The useful automation task is triage, not replacing the support agent.

Pipeline

Models

The prototype used transformer models for two different steps: sentiment filtering to decide whether a tweet looked like a complaint, and zero-shot classification to route complaints into broad categories without training a new classifier from scratch.

StagePurpose
CleaningNormalize tweet text and remove duplicate records.
SentimentFilter for likely negative or complaint-like messages.
Zero-shot routingAssign broad issue categories from natural-language labels.
Response templateGenerate a structured support reply draft.

Repository boundaries

The public repository excludes API keys and tweet data. It keeps the reusable processing logic and documents which data/configuration must stay local.

Automated Twitter Complaint Chatbot

A prototype that filters likely airline complaints using sentiment analysis, routes them into complaint categories with zero-shot classification, and returns templated support responses.

Problem

Social support teams receive high-volume, noisy tweets. The useful automation target is triage: identify complaints, classify the issue, and suggest a response while leaving final responsibility to a human operator.

Approach

The pipeline cleans tweet JSON, imports data into MongoDB, deduplicates records, scores sentiment using a Twitter RoBERTa model, and uses zero-shot classification to route complaints into broad and specific categories.

raw tweet JSON -> cleaning + MongoDB import
              -> duplicate removal
              -> sentiment score
              -> zero-shot issue category
              -> response template

Boundaries

  • No API credentials are included.
  • No tweet dataset is included.
  • Local MongoDB URI is configurable.
  • Models download from Hugging Face at runtime.

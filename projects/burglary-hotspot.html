Burglary Hotspot Forecasting

A spatio-temporal burglary hotspot prediction pipeline that transforms London crime data into monthly grid-based risk maps for patrol allocation.

Problem

The task was to forecast where burglaries are likely to occur in the next month. A useful model needed spatial structure, temporal history, and a practical output format that could be mapped and interpreted.

Approach

The pipeline filters burglary records, projects coordinates into a metric CRS, creates 500-meter grid cells, aggregates incidents per month, and builds lag features such as previous-month activity, rolling three-month activity, and neighbor activity. A ConvLSTM-style model learns from short sequences of spatial feature maps.

police.uk data -> burglary filter -> projected grid
              -> monthly labels + lag features
              -> ConvLSTM sequence model
              -> threshold tuning
              -> GeoJSON-ready hotspot outputs

Leadership

I also chaired the multidisciplinary project team, setting agendas and planning task ownership. This project is one of the clearest examples of combining modeling, implementation, and coordination.

Stack

  • Python, pandas, NumPy
  • GeoPandas and Shapely
  • PyTorch ConvLSTM
  • Precision, recall, F1, confusion matrix tracking

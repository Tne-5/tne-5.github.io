Burglary Hotspot Forecasting

A London residential burglary project that turns monthly crime records into 500m grid predictions and patrol-demand estimates.

Task

The project asked how police demand could be estimated automatically so limited patrol capacity could be directed toward high-risk residential burglary areas.

Approach

The pipeline groups Police.uk burglary records by month, enriches them with weather and socioeconomic features, projects locations to a metric CRS, builds 500m grid labels, and trains a ConvLSTM on three-month spatial sequences.

crime records -> monthly LSOA grouping -> external features
              -> 500m grid tensors -> ConvLSTM
              -> thresholded hotspots -> residential-area allocation

Result

The report-level evaluation showed 82.96% accuracy, 0.8716 recall for burglary cells, 0.7258 F1 for burglary cells, and 0.8374 weighted F1. The prototype also included a Streamlit-style patrol allocation workflow.

Stack

  • Python, pandas, GeoPandas, Shapely
  • PyTorch ConvLSTM
  • 500m spatial grid classification
  • Residential land-use filtering
  • PuLP-style allocation logic
